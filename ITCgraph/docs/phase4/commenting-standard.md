# ITCgraph Phase 4: 开发注释规范

## 1. 适用范围

适用于 `ITCgraph` 的新增/改动代码。目标是让 Step3 绘图与 Step2 桥接逻辑具备可审查注释。
策略：仅对新增/改动文件强制，不要求一次性重写历史文件。

## 2. 注释类型与字段

### 2.1 模块头注释（必填）

字段固定：

1. `模块职责`
2. `依赖`
3. `对外接口`
4. `副作用`
5. `变更历史`

### 2.2 关键算法注释（核心逻辑推荐）

字段固定：

1. `算法目标`
2. `输入约束`
3. `数值稳定策略`
4. `失败回退`
5. `复杂度`

### 2.3 I/O 契约注释（边界函数必填）

字段固定：

1. `输入来源`
2. `字段/类型`
3. `单位`
4. `空值策略`
5. `输出保证`

### 2.4 错误语义注释（宿主入口必填）

字段固定：

1. `错误码/类别`
2. `触发条件`
3. `用户可见性`
4. `日志级别`
5. `恢复动作`

## 3. 最小覆盖要求（Phase 4）

1. `app.R`：`[MODULE_HEADER]` + `[ERROR_SEMANTICS]`
2. `R/bridge_plot_helpers.R`：`[MODULE_HEADER]` + `[IO_CONTRACT]`

## 4. 反例

1. 注释仅描述“做了什么”，没有输入输出约束。
2. 桥接 payload 处理函数未写字段和空值策略。
3. 启动错误路径没有可见性与恢复动作说明。
