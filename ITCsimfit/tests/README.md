# 单元测试说明

## 测试文件

- `test_core_logic.R` - 核心逻辑模块的单元测试

## 运行测试

### 方法 1: 从项目根目录运行（推荐）

```r
Rscript tests/test_core_logic.R
```

### 方法 2: 在 RStudio 中运行

```r
# 在 RStudio 中打开测试文件，然后运行：
source("tests/test_core_logic.R")
```

### 方法 3: 在 R 控制台中运行

```r
# 确保在项目根目录
setwd("项目路径")
source("tests/test_core_logic.R")
```

## 测试内容

### 测试 1: solve_equi_modular - 基础 1:1 结合
- 测试基本的 1:1 结合反应
- 验证质量守恒
- 验证返回值的合理性

### 测试 2: solve_equi_modular - 边界情况
- 测试 G_tot = 0 的情况
- 测试 H_tot = 0 的情况
- 验证边界处理的正确性

### 测试 3: solve_equi_modular - 复杂模型（D 路径）
- 测试包含 D 路径的复杂模型
- 验证多路径反应的质量守恒

### 测试 4: run_sim_modular - 基础模拟
- 测试基本的模拟功能
- 验证返回数据框的结构
- 验证数据的合理性

### 测试 5: run_sim_modular - 预注射情况
- 测试预注射功能
- 验证预注射对结果的影响

### 测试 6: 参数验证
- 测试参数验证机制
- 验证边界值的处理

## 测试框架

当前使用简单的自定义断言函数。如果需要更强大的测试框架，可以考虑使用 `testthat` 包：

```r
install.packages("testthat")
```

然后可以使用 `testthat` 的测试函数重写测试。
