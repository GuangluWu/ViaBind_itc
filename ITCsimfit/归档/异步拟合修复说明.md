# 异步拟合修复说明

## 🐛 问题描述

**错误信息：**
```
拟合无法启动：模拟计算出错: Can't access reactive value 'active_paths' outside of reactive consumer.
ℹ Do you need to wrap inside reactive() or observe()?
```

## 🔍 问题原因

在异步拟合中，`future()` 会在**后台进程**中执行代码。Shiny 的 `input$` 对象只能在**响应式上下文**（reactive context）中访问，在后台进程中无法访问。

**问题代码：**
```r
fit_promise <- future({
  perform_fitting_sync(
    ...
    active_paths = input$active_paths,  # ❌ 在后台进程中无法访问 input$
    enable_error_analysis = input$enable_error_analysis  # ❌ 同样的问题
  )
})
```

## ✅ 解决方案

在调用 `future()` **之前**，先将所有 `input$` 值提取到局部变量中，然后传递给后台进程。

**修复后的代码：**
```r
# [重要] 在 future() 之前提取所有 input$ 值
active_paths_val <- input$active_paths
enable_error_analysis_val <- if(!is.null(input$enable_error_analysis)) input$enable_error_analysis else FALSE

fit_promise <- future({
  perform_fitting_sync(
    ...
    active_paths = active_paths_val,  # ✅ 使用局部变量
    enable_error_analysis = enable_error_analysis_val  # ✅ 使用局部变量
  )
})
```

修复后，异步拟合应该可以正常工作：

3. ✅ 界面保持响应

## 💡 重要提示

**在 Shiny 中使用 `future()` 时：**
- ❌ **不要**在 `future()` 内部直接访问 `input$`、`output$` 或 `values$`
- ✅ **应该**在调用 `future()` 之前提取所有需要的值
- ✅ 将提取的值作为参数传递给后台进程中的函数

**需要提取的常见值：**
- `input$active_paths`
- `input$enable_error_analysis`
- `input$fit_params`
- `input$fit_data_range`
- 其他在后台进程中需要的 `input$` 值
