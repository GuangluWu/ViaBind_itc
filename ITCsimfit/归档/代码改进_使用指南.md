## 新增的模块

### 1. `R/constants.R` - 常量定义

包含所有项目常量：
- **参数边界** (`PARAM_BOUNDS`)：logK, H, fH/fG, Offset, V_init
- **数值计算常量** (`EPSILON`, `EPSILON_LOG`, `SOLVER`)
- **优化算法参数** (`DE_OPTIM`, `LBFGS_OPTIM`)
- **UI默认值** (`UI_DEFAULTS`, `NOTIFICATION_DURATION`)
- **文件路径** (`FILE_PATHS`)
- **错误代码** (`ERROR_CODES`)

### 2. `R/utils.R` - 工具函数

提供通用工具函数：
- **错误处理**：`handle_error()`, `safe_execute()`
- **输入验证**：`safe_input()`, `safe_numeric()`, `validate_dataframe()`
- **数值计算**：`safe_log()`, `safe_divide()`, `is_near_zero()`
- **其他工具**：进度条、缓存、格式化、性能测量等

# server.R 第965-977行
lower_b <- par_vec; upper_b <- par_vec

## 使用统一错误处理

### ❌ 改进前（不一致的错误处理）

```r
# 示例1：简单返回 NULL
tryCatch({
  data <- read.csv(file_path)
}, error = function(e) {
  return(NULL)
})

# 示例2：显示通知但没有日志
tryCatch({
  result <- fit_data(params)
}, error = function(e) {
  showNotification(e$message, type = "error")
  return(NULL)
})

# 示例3：只输出到控制台
tryCatch({
  sim_result <- calculate_simulation(...)
}, error = function(e) {
  print(paste("Error:", e$message))
  return(NULL)
})
```

### ✅ 改进后（统一错误处理）

```r
# 使用 handle_error
tryCatch({
  data <- read.csv(file_path)
}, error = function(e) {
  handle_error(e, 
              context = "读取CSV文件", 
              lang_val = lang())
  return(NULL)
})

# 或使用 safe_execute（更简洁）
data <- safe_execute({
  read.csv(file_path)
}, context = "读取CSV文件", default = NULL, lang_val = lang())

# 拟合计算
result <- safe_execute({
  fit_data(params)
}, context = "拟合计算", default = NULL, show_error = TRUE, lang_val = lang())

# 模拟计算
sim_result <- safe_execute({
  calculate_simulation(...)
}, context = "模拟计算", default = NULL, lang_val = lang())
```

## 代码迁移示例

### 示例1：改进 `server.R` 中的拟合函数

#### 改进前：

```r
perform_fitting_sync <- function() {
  # 硬编码的边界定义
  lower_b <- par_vec; upper_b <- par_vec
  for(nm in names(par_vec)) {
    if(grepl("logK", nm)) { lower_b[nm] <- 1; upper_b[nm] <- 9 }
    if(grepl("H", nm))    { lower_b[nm] <- -15000; upper_b[nm] <- 5000 }
    # ... 更多硬编码
  }
  
  # 硬编码的优化参数
  pop_size <- max(10 * length(par_vec), 50)
  
  # 简单的错误处理
  tryCatch({
    de_res <- DEoptim(fn = cost_fn,
                     lower = lower_b,
                     upper = upper_b,
                     control = list(itermax = 100, NP = pop_size))
  }, error = function(e) {
    showNotification(paste("拟合失败:", e$message), type = "error")
    return(NULL)
  })
}
```

#### 改进后：

```r
perform_fitting_sync <- function() {
  # 使用常量和辅助函数
  bounds <- get_parameter_bounds(names(par_vec), v_inj = v_inj_val)
  
  # 计算种群大小
  pop_size <- max(
    DE_OPTIM$pop_size_multiplier * length(par_vec),
    DE_OPTIM$min_pop_size
  )
  pop_size <- min(pop_size, DE_OPTIM$max_pop_size)
  
  # 使用统一错误处理
  de_res <- safe_execute({
    DEoptim(fn = cost_fn,
           lower = bounds$lower,
           upper = bounds$upper,
           control = list(
             itermax = DE_OPTIM$max_iter,
             NP = pop_size,
             F = DE_OPTIM$F,
             CR = DE_OPTIM$CR,
             trace = DE_OPTIM$trace,
             strategy = DE_OPTIM$strategy
           ))
  }, 
  context = "DE优化拟合",
  error_code = ERROR_CODES$OPTIMIZATION_FAILED,
  lang_val = lang())
  
  if(is.null(de_res)) {
    return(NULL)
  }
  
  # 记录拟合完成
  log_info("拟合成功完成", context = "perform_fitting_sync")
  
  return(de_res)
}
```

## 最佳实践

### 1. 总是使用常量而不是魔法数字

```r
# ❌ 不好
if(x < 1e-20) { ... }

# ✅ 好
if(is_near_zero(x, EPSILON)) { ... }
```

### 2. 使用统一的错误处理

```r
# ❌ 不好
tryCatch({ ... }, error = function(e) { print(e); NULL })

# ✅ 好
safe_execute({ ... }, context = "操作描述", lang_val = lang())
```

### 3. 验证输入

```r
# ❌ 不好
value <- input$slider

# ✅ 好
value <- safe_numeric(input$slider, default = DEFAULT_PARAMS$logK, 
                     min = PARAM_BOUNDS$logK["lower"],
                     max = PARAM_BOUNDS$logK["upper"])
```

### 4. 记录重要操作

```r
# 记录信息
log_info("开始拟合计算", context = "fitting")

# 记录警告
log_warning("数据点较少，拟合结果可能不准确", context = "fitting")

# 使用 time_it 测量性能
result <- time_it({
  expensive_calculation()
}, label = "昂贵的计算", log_result = TRUE)
```

### 5. 使用辅助函数简化代码

```r
# ❌ 不好 - 重复的边界定义
lower_b <- numeric(length(params))
upper_b <- numeric(length(params))
for(i in seq_along(params)) {
  if(params[i] == "logK1") { lower_b[i] <- 1; upper_b[i] <- 9 }
  # ... 更多重复
}

# ✅ 好 - 使用辅助函数
bounds <- get_parameter_bounds(params, v_inj = v_inj_val)
```

## 迁移清单

在迁移现有代码时，按以下顺序进行：

- [x] 1. 创建 `R/constants.R` 和 `R/utils.R`
- [x] 2. 更新 `global.R` 加载新模块
- [ ] 3. 替换参数边界定义
  - [ ] `server.R` 中的 `perform_fitting_sync` 函数
  - [ ] 其他使用参数边界的地方
- [ ] 4. 替换硬编码的数值常量
  - [ ] `1e-20` → `EPSILON`
  - [ ] `1e-15` → `EPSILON_LOG`
  - [ ] `-15000, 5000` → `PARAM_BOUNDS$H`
- [ ] 5. 统一错误处理
  - [ ] 替换所有 `tryCatch` 使用 `safe_execute`
  - [ ] 添加上下文和错误代码
- [ ] 6. 改进输入验证
  - [ ] 使用 `safe_input()` 和 `safe_numeric()`
  - [ ] 使用 `validate_dataframe()`
- [ ] 7. 替换 UI 硬编码值
  - [ ] 使用 `UI_DEFAULTS` 常量
- [ ] 8. 测试所有改动
  - [ ] 运行应用，测试基本功能
  - [ ] 测试错误处理（故意触发错误）
  - [ ] 检查日志文件
