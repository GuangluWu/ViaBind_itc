### 1. 核心模块开发

#### `R/performance.R` (679 行)

**主要功能：**

- ✅ **性能监控配置系统**
  - `PERF_CONFIG`：全局配置（启用/禁用、日志级别、阈值等）
  - `set_perf_config()`：动态修改配置

- ✅ **核心监控函数**
  - `perf_monitor()`：监控单个操作的执行时间和内存使用
  - `perf_monitor_batch()`：监控多步骤操作流程
  - `with_perf_monitor()`：函数包装器（装饰器模式）

- ✅ **性能记录管理**
  - `add_perf_record()`：添加性能记录到全局存储
  - `get_perf_records()`：查询性能记录（支持类别、耗时、数量筛选）
  - `clear_perf_records()`：清除性能记录

- ✅ **智能日志系统**
  - `should_log()`：根据配置判断是否记录
  - `log_perf_record()`：记录性能日志到文件
  - 自动标记慢操作（SLOW）和警告（WARNING）

- ✅ **性能分析和报告**
  - `generate_perf_report()`：生成详细的性能统计报告
  - `analyze_performance()`：识别性能瓶颈并提供优化建议

- ✅ **辅助工具**
  - `is_slow_operation()`：判断操作是否过慢
  - `perf_warning()`：性能警告

## 🎯 核心特性

### 1. 灵活的配置系统

```r
set_perf_config(
  enabled = TRUE,              # 全局开关
  log_level = "detailed",      # 日志级别
  slow_threshold = 1.0,        # 慢操作阈值（秒）
  warning_threshold = 5.0,     # 警告阈值（秒）
  auto_log_slow = TRUE,        # 自动记录慢操作
  console_output = FALSE,      # 控制台输出
  max_records = 1000           # 最大记录数
)
```

### 2. 强大的监控能力

**单个操作监控：**
```r
result <- perf_monitor({
  perform_calculation()
}, label = "计算", category = "calculation")

# 获取结果和性能指标
data <- result$result
time <- result$elapsed        # 耗时（秒）
memory <- result$memory_used  # 内存使用（MB）
```

**多步骤监控：**
```r
results <- perf_monitor_batch(
  steps = list(
    list(expr = quote({step1()}), label = "步骤1"),
    list(expr = quote({step2()}), label = "步骤2")
  ),
  overall_label = "完整流程",
  category = "workflow"
)

# 查看各步骤耗时
print(results$summary$step_timings)
```

**函数包装：**
```r
my_func_monitored <- with_perf_monitor(
  my_func,
  label = "我的函数",
  category = "calculation"
)
```

### 3. 智能日志系统

**自动记录：**
- 慢操作（超过 `slow_threshold`）
- 错误操作
- 根据日志级别过滤

**日志格式：**
```
[PERF] 2026-01-27 10:15:23 | fitting | ITC拟合 | 4.502s | Mem: +12.3 MB | SLOW | WARNING
```

### 4. 性能分析和报告

**生成报告：**

```r
generate_perf_report()
# 输出：总记录数、按类别统计、最慢的操作
```

**分析瓶颈：**

## 📈 应用场景

### 1. ITC 数据拟合

```r
# 监控完整的拟合流程
result <- perf_monitor_batch(
  steps = list(
    list(expr = quote({validate_data()}), label = "数据验证"),
    list(expr = quote({global_optimize()}), label = "全局优化"),
    list(expr = quote({local_refine()}), label = "局部精化"),
    list(expr = quote({error_analysis()}), label = "误差分析")
  ),
  overall_label = "ITC拟合",
  category = "fitting"
)
```

**价值：**
- 识别拟合过程中的性能瓶颈（如全局优化过慢）
- 优化迭代次数和种群大小
- 平衡精度和速度

### 2. 大数据文件处理

```r
# 监控数据读取和处理
result <- perf_monitor({
  data <- read.csv(large_file)
  processed <- process_data(data)
  processed
}, 
label = "大文件处理",
category = "data_io")
```

**价值：**
- 发现文件读取瓶颈
- 优化数据处理算法
- 考虑分批处理或缓存

### 3. 批量模拟

```r
# 监控批量模拟
results <- lapply(param_sets, function(params) {
  perf_monitor({
    simulate_itc(params)
  },
  label = sprintf("模拟(logK=%.1f)", params$logK),
  category = "simulation")$result
})

# 查看所有模拟的耗时分布
slow_sims <- get_perf_records(
  category = "simulation",
  min_elapsed = 1.0
)
```

**价值：**
- 识别慢参数组合
- 优化模拟算法
- 并行化批量模拟

### 4. 用户交互响应

```r
# 监控 Shiny 交互
observeEvent(input$action_button, {
  result <- perf_monitor({
    # 处理用户请求
    process_user_request()
  },
  label = "用户操作处理",
  category = "user_interaction")
  
  # 响应时间超过2秒，提示用户
  if (result$elapsed > 2.0) {
    showNotification(
      "操作耗时较长，建议优化参数",
      type = "warning"
    )
  }
})
```

**价值：**
- 保证用户体验（< 2秒响应）
- 识别影响交互的操作
- 优化关键路径
