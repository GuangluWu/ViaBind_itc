## 🔍 具体改进示例

### 示例1：参数边界定义

**改进前（重复、硬编码）：**
```r
# ui.R
sliderInput("logK1", "logK1", 1, 9, 7.0, ...)

# server.R
lower_b[nm] <- 1; upper_b[nm] <- 9

# core_logic.R（不明确）
```

**改进后（统一、常量）：**
```r
# constants.R（集中定义）
PARAM_BOUNDS <- list(logK = c(lower = 1, upper = 9))
DEFAULT_PARAMS <- list(logK = 5)

# ui.R
sliderInput("logK1", "logK1", 
           min = PARAM_BOUNDS$logK["lower"],
           max = PARAM_BOUNDS$logK["upper"],
           value = DEFAULT_PARAMS$logK, ...)

# server.R
bounds <- get_parameter_bounds(names(par_vec), v_inj)
lower_b <- bounds$lower; upper_b <- bounds$upper

# 三个模块完全一致！
```

### 示例2：错误处理

**改进前（不统一）：**
```r
# 各种不同的错误处理方式
tryCatch({ ... }, error = function(e) NULL)
tryCatch({ ... }, error = function(e) print(e))
if(error) showNotification(...)
```

**改进后（统一）：**
```r
# 统一使用 safe_execute
safe_execute({
  operation()
}, context = "操作描述", lang_val = lang())

# 自动：
# - 捕获错误
# - 记录日志
# - 显示通知
# - 返回默认值
```

### 示例3：数值计算

**改进前（魔法数字）：**
```r
if(G_tot <= 1e-20) ...
if(x < 1e-15) ...
Ks <- pmax(Ks, 1e-20)
log(pmax(x, 1e-15))
```

**改进后（语义化）：**
```r
if(is_near_zero(G_tot, EPSILON)) ...
if(is_near_zero(x, EPSILON_LOG)) ...
Ks <- pmax(Ks, EPSILON)
safe_log(x, EPSILON_LOG)
```
