# UI 限制验证 - 最小注射次数

## 修改日期
2026-01-27

## 问题描述
单次注射（n_inj = 1）在向量化计算中存在边界问题：
- 错误：`replacement has length zero`
- 原因：某些向量操作（如 `diff()`、`[-n_inj]`）需要至少 2 个元素
- 影响范围：极少（实际 ITC 实验很少只做 1 针）

## 解决方案

### 1. UI 限制
在 `ui.R` 中限制最小注射次数为 2：

```r
numericInput("n_inj", "N Inj", 
            value = UI_DEFAULTS$n_inj_default,
            min = 2,  # 修复：限制最小注射次数为 2
            max = UI_DEFAULTS$n_inj_max)
```

**修改位置**: `ui.R` 第 570 行

### 2. 常量定义
在 `R/constants.R` 中添加最小值常量：

```r
UI_DEFAULTS <- list(
  n_inj_min = 2,              # 最小注射次数（避免单次注射的向量化边界问题）
  n_inj_default = 26,         # 默认注射次数
  n_inj_max = 50,             # 最大注射次数
  ...
)
```

**修改位置**: `R/constants.R` 第 117 行

## 技术说明

### 为什么需要至少 2 次注射？

向量化代码中的关键操作：

```r
# 计算差分热量（需要前一针的热量）
Q_prev_vec <- c(Q_prev * disp_ratio, Q_total_vec[-n_inj] * disp_ratio)
#                                     ^^^^^^^^^^^^^^^^
#                                     这个操作需要至少 2 个元素
```

当 `n_inj = 1` 时：
- `Q_total_vec[-1]` 尝试移除第 1 个元素
- 结果是一个长度为 0 的向量
- 导致 `c(Q_prev * disp_ratio, numeric(0))` 操作异常

类似的问题出现在：
```r
# 计算 Ratio 变化率
diff(result$Ratio)  # 需要至少 2 个点才能计算差分
```

### 替代方案（未采用）

```r
if (n_inj == 1) {
  # 特殊处理逻辑

} else {
  # 正常向量化逻辑

}
```

**不采用原因**：
- 增加代码复杂度
- 单次注射在实际应用中极少出现
- UI 限制是更简洁的解决方案

#### 方案 2: 保持 min = 1，但添加运行时验证
```r
if (n_inj < 2) {

}
```

**不采用原因**：
- 用户可能会困惑（为什么可以输入但不能运行）
