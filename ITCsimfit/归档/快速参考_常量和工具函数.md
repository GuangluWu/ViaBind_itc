# 快速参考：常量和工具函数

## 🚀 最常用的功能

### 1. 参数边界

```r
# 获取单个参数边界
bounds <- get_param_bound("logK1")
# 返回: c(lower = 1, upper = 9)

# 获取多个参数边界
param_names <- c("logK1", "H1", "fH", "V_init")
bounds <- get_parameter_bounds(param_names, v_inj = 0.01)
# 返回: list(lower = c(...), upper = c(...))
```

### 2. 错误处理

```r
# 安全执行（最常用）
result <- safe_execute({
  your_risky_operation()
}, context = "操作描述", default = NULL, lang_val = lang())

# 完整错误处理
tryCatch({
  result <- operation()
}, error = function(e) {
  handle_error(e, context = "操作名称", lang_val = lang())
  return(NULL)
})
```

### 3. 输入验证

```r
# 安全获取输入（处理 NULL/NA）
value <- safe_input(input$slider, default = 1.0)

# 安全获取数值（带范围检查）
value <- safe_numeric(input$slider, 
                     default = 1.0, 
                     min = 0, 
                     max = 10)

# 验证数据框
is_valid <- validate_dataframe(df, 
                              required_cols = c("Q", "inj_idx"),
                              min_rows = 5)
```

### 4. 数值计算

```r
# 安全对数（防止 log(0)）
log_val <- safe_log(x)  # 自动使用 EPSILON_LOG

# 安全除法（防止除零）
result <- safe_divide(numerator, denominator)

# 检查是否接近零
if(is_near_zero(value, EPSILON)) {
  # 处理接近零的情况
}
```

### 参数边界

| 参数类型 | 下界 | 上界 | 访问方式 |
|---------|------|------|---------|
| logK | 1 | 9 | `PARAM_BOUNDS$logK` |
| H (cal/mol) | -15000 | 5000 | `PARAM_BOUNDS$H` |
| fH/fG | 0.5 | 1.5 | `PARAM_BOUNDS$fH_fG` |
| Offset (cal) | -1500 | 1500 | `PARAM_BOUNDS$Offset` |
| V_init | 0 | 1×V_inj | `PARAM_BOUNDS$V_init_multiplier` |

### 数值常量

| 常量 | 值 | 用途 |
|------|-----|------|
| `EPSILON` | 1e-20 | 通用极小值阈值 |
| `EPSILON_LOG` | 1e-15 | 对数计算阈值 |
| `EPSILON_TOLERANCE` | 1e-7 | 收敛容差 |

## 📖 常见使用模式

### 模式1：拟合函数中设置参数边界

```r
# ❌ 旧方式
lower_b <- par_vec; upper_b <- par_vec
for(nm in names(par_vec)) {
  if(grepl("logK", nm)) { lower_b[nm] <- 1; upper_b[nm] <- 9 }
  if(grepl("H", nm))    { lower_b[nm] <- -15000; upper_b[nm] <- 5000 }
  # ...
}

# ✅ 新方式
bounds <- get_parameter_bounds(names(par_vec), v_inj = v_inj_val)
lower_b <- bounds$lower
upper_b <- bounds$upper
```

### 模式2：DE 优化配置

```r
# ❌ 旧方式
pop_size <- max(10 * length(par_vec), 50)
de_res <- DEoptim(fn = fn, lower = lb, upper = ub,
                  control = list(itermax = 100, NP = pop_size, trace = FALSE))

# ✅ 新方式
pop_size <- max(DE_OPTIM$pop_size_multiplier * length(par_vec),
                DE_OPTIM$min_pop_size)
pop_size <- min(pop_size, DE_OPTIM$max_pop_size)

de_res <- safe_execute({
  DEoptim(fn = fn, lower = lb, upper = ub,
          control = list(
            itermax = DE_OPTIM$max_iter,
            NP = pop_size,
            F = DE_OPTIM$F,
            CR = DE_OPTIM$CR,
            trace = DE_OPTIM$trace,
            strategy = DE_OPTIM$strategy
          ))
}, context = "DE优化", lang_val = lang())
```

### 模式3：输入验证

```r
# ❌ 旧方式
n_inj <- input$n_inj
if(is.null(n_inj)) n_inj <- 26
if(n_inj < 1) n_inj <- 1
if(n_inj > 50) n_inj <- 50

# ✅ 新方式
n_inj <- safe_numeric(input$n_inj,
                     default = UI_DEFAULTS$n_inj_default,
                     min = 1,
                     max = UI_DEFAULTS$n_inj_max)
```

### 模式4：数据验证

```r
# ❌ 旧方式
if(is.null(data) || !is.data.frame(data)) {
  showNotification("无效数据", type = "error")
  return(NULL)
}
if(!"Q" %in% colnames(data)) {
  showNotification("缺少 Q 列", type = "error")
  return(NULL)
}

# ✅ 新方式
if(!validate_dataframe(data, 
                      required_cols = c("Q", "inj_idx"),
                      min_rows = DATA_VALIDATION$min_data_points)) {
  handle_error(simpleError("数据验证失败"),
              context = "数据加载",
              error_code = ERROR_CODES$DATA_INVALID,
              lang_val = lang())
  return(NULL)
}
```

### 模式5：数值稳定性检查

```r
# ❌ 旧方式
if(G_tot <= 1e-20) return(c(0, H_tot, 0))
log_val <- log(pmax(x, 1e-15))

# ✅ 新方式
if(is_near_zero(G_tot, EPSILON)) return(c(0, H_tot, 0))
log_val <- safe_log(x, EPSILON_LOG)
```

## 🎯 迁移检查清单

在迁移每个函数时，检查以下项目：

- [ ] **硬编码的数值** → 使用常量
  - [ ] `-15000, 5000` → `PARAM_BOUNDS$H`
  - [ ] `1e-20` → `EPSILON`
  - [ ] `1e-15` → `EPSILON_LOG`
  - [ ] `26` (注射次数) → `UI_DEFAULTS$n_inj_default`

- [ ] **参数边界定义** → 使用 `get_parameter_bounds()`

- [ ] **错误处理** → 使用 `safe_execute()` 或 `handle_error()`

- [ ] **输入验证** → 使用 `safe_input()` 或 `safe_numeric()`

- [ ] **数据验证** → 使用 `validate_dataframe()`

- [ ] **数值计算** → 使用 `safe_log()` 或 `safe_divide()`

- [ ] **日志记录** → 添加 `log_info()` 或 `log_warning()`

## 💡 提示和技巧

### 1. 调试模式

```r
# 在 global.R 中设置
DEBUG_MODE <- TRUE  # 启用调试输出

# 在代码中使用
debug_print("调试信息", prefix = "[DEBUG]")
print_object_info(my_object, "对象名称")
```

### 2. 性能分析

```r
# 测量关键操作耗时
result <- time_it({
  simulation_result <- calculate_simulation(...)
}, label = "模拟计算", log_result = TRUE)
# 自动记录到日志：模拟计算 completed in 0.123 seconds
```

### 3. 错误日志

所有通过 `handle_error()` 处理的错误会自动记录到：
- `error.log` - 错误和警告
- `session.log` - 信息日志

### 4. 缓存使用

```r
# 生成缓存键
cache_key <- generate_cache_key(param1, param2, param3)

# 检查缓存
if(cache_hit(cached_result, cached_key, cache_key)) {
  result <- cached_result
} else {
  result <- expensive_calculation()
  cached_result <- result
  cached_key <- cache_key
}
```
