## 📝 具体改进内容

### 1. 替换极小值边界检查（第61-62行）

#### ❌ 改进前
```r
# 0. 边界处理
if(G_tot <= 1e-20) return(c(0, H_tot, 0))
if(H_tot <= 1e-20) return(c(G_tot, 0, 0))
```

**问题：**
- 硬编码的魔法数字 `1e-20`
- 与其他地方使用的阈值不一致
- 难以统一调整

#### ✅ 改进后
```r
# 0. 边界处理 - 使用常量定义的极小值阈值
if(is_near_zero(G_tot, EPSILON)) return(c(0, H_tot, 0))
if(is_near_zero(H_tot, EPSILON)) return(c(G_tot, 0, 0))
```

**改进效果：**
- ✅ 使用 `EPSILON` 常量（值为 1e-20）
- ✅ 使用 `is_near_zero()` 函数，语义更清晰
- ✅ 统一的阈值管理
- ✅ 易于全局调整

### 2. 替换平衡常数的保护（第111-112行）

#### ❌ 改进前
```r
# 准备常数
Ks <- c(p$K1, p$K2, p$K3, p$K4, p$K5)
Ks <- pmax(Ks, 1e-20) # 防止有某个K变成0或者负数，无法计算lnK
lnKs <- log(Ks)
```

**问题：**
- 硬编码的 `1e-20`
- 直接使用 `log()`，可能在边界情况下有问题

#### ✅ 改进后
```r
# 准备常数 - 使用常量定义的极小值阈值
Ks <- c(p$K1, p$K2, p$K3, p$K4, p$K5)
Ks <- pmax(Ks, EPSILON) # 防止有某个K变成0或者负数，无法计算lnK
lnKs <- safe_log(Ks, EPSILON_LOG)
```

**改进效果：**
- ✅ 使用 `EPSILON` 常量保护 K 值
- ✅ 使用 `safe_log()` 函数，防止 log(0) 错误
- ✅ 使用 `EPSILON_LOG` 作为对数计算的阈值
- ✅ 双重保护，更加稳健

### 4. 替换默认参数值（第307-308行和第366-367行）

#### ❌ 改进前
```r
# [关键修复] 计算预注射后的初始热量基线 (Q_prev)
safe_K <- function(val) if(is.null(val) || is.na(val)) 1e7 else 10^val
safe_H <- function(val) if(is.null(val) || is.na(val)) -5000 else val
```

**问题：**
- 硬编码的默认值 `1e7` 和 `-5000`
- 与其他地方的默认值不一致
- 魔法数字，意义不明确

#### ✅ 改进后
```r
# [关键修复] 计算预注射后的初始热量基线 (Q_prev)
# 使用常量定义的默认值
safe_K <- function(val) if(is.null(val) || is.na(val)) 10^DEFAULT_PARAMS$logK else 10^val
safe_H <- function(val) if(is.null(val) || is.na(val)) DEFAULT_PARAMS$H else val
```

**改进效果：**
- ✅ 使用 `DEFAULT_PARAMS` 常量
- ✅ 统一的默认值管理
- ✅ 语义清晰（logK=5 对应 K=1e5，H=-6000）
- ✅ 易于全局调整

### 5. 替换其他硬编码值（第80行、第320行、第360行、第493行）

#### ❌ 改进前
```r
K <- max(K, 1e-3) # 防止 K=0

sol_pre <- solve_equi_modular(G_t_real, H_t_real, temp_lin_p, active_paths, c(1e-12, H_t_real))

current_guess <- c(1e-12, H_t_real) # 更新猜测值 (基于真实浓度)
```

**问题：**

```r
sol_pre <- solve_equi_modular(G_t_real, H_t_real, temp_lin_p, active_paths, c(EPSILON_LOG, H_t_real))

current_guess <- c(EPSILON_LOG, H_t_real) # 更新猜测值，使用常量定义的极小值

K <- max(K1, EPSILON)  # 防止 K=0，使用常量定义的极小值阈值
```

**改进效果：**
- ✅ 统一使用 `EPSILON` 和 `EPSILON_LOG`
- ✅ 语义一致，易于理解
- ✅ 便于全局调整


### 数值阈值调整

如需调整数值阈值，只需修改 `R/constants.R`：
```r
# 在 constants.R 中修改
EPSILON <- 1e-20        # 通用极小值阈值
EPSILON_LOG <- 1e-15    # 对数计算阈值
```

所有使用这些常量的地方会自动生效。

### 默认参数调整

如需调整默认参数，修改 `R/constants.R`：
```r

  logK = 5,      # 默认 logK = 5 (K = 10^5)
  H = -6000,     # 默认焓变 -6000 cal/mol
  # ...
)
```
