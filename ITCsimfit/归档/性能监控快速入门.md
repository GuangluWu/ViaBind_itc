# 性能监控快速入门 ⚡

## 🚀 5分钟快速上手

### 第一步：配置性能监控

在 `global.R` 中已自动加载性能监控模块。您可以在 `server.R` 开头配置参数：

```r
# 在 server.R 开头添加
set_perf_config(
  enabled = TRUE,           # 启用性能监控
  slow_threshold = 1.0,     # 操作超过1秒视为慢操作
  console_output = TRUE     # 在控制台显示日志（开发时）
)
```

### 第二步：监控关键操作

#### 方式1：包装代码块（推荐）

```r
# 在 observeEvent 或函数中
result <- perf_monitor({
  # 你的代码
  perform_fitting(data, params)
}, 
label = "数据拟合",        # 给操作起个名字
category = "fitting")      # 分类：fitting, simulation, data_io 等

# 使用结果
fitted_data <- result$result
time_used <- result$elapsed  # 查看耗时
```

#### 方式2：监控多步骤流程

```r
results <- perf_monitor_batch(
  steps = list(
    list(expr = quote({ step1() }), label = "步骤1"),
    list(expr = quote({ step2() }), label = "步骤2"),
    list(expr = quote({ step3() }), label = "步骤3")
  ),
  overall_label = "完整流程",
  category = "workflow"
)

# 查看各步骤耗时
print(results$summary$step_timings)
```

### 第三步：查看性能报告

在 R 控制台运行：

```r
# 生成性能报告
generate_perf_report()

# 或保存到文件
generate_perf_report(output_file = "性能报告.txt")

# 分析性能瓶颈
analyze_performance()
```

## 📍 常见使用场景

### 场景1：监控数据拟合

```r
observeEvent(input$run_fitting, {
  result <- perf_monitor({
    fit_data <- perform_fitting_sync(
      data = experiment_data(),
      params = get_initial_params()
    )
    fit_data
  }, 
  label = "ITC数据拟合", 
  category = "fitting")
  
  # 根据耗时给用户反馈
  if (result$elapsed > 5) {
    showNotification(
      sprintf("拟合完成，耗时 %.1f 秒", result$elapsed),
      type = "warning"
    )
  }
  
  # 更新结果
  fitted_data(result$result)
})
```

### 场景2：监控文件导入

```r
observeEvent(input$file_upload, {
  result <- perf_monitor({
    file_path <- input$file_upload$datapath
    data <- read.csv(file_path)
    validate_dataframe(data, required_cols = c("V_inj", "dQ_obs"))
  },
  label = "数据文件导入",
  category = "data_io")
  
  if (!is.null(result$result)) {
    uploaded_data(result$result)
    showNotification(
      sprintf("数据导入成功 (%.2f秒)", result$elapsed),
      type = "message"
    )
  }
})
```

### 场景3：监控模拟计算

```r
observeEvent(input$run_simulation, {
  result <- perf_monitor({
    simulate_itc_curve(
      params = list(logK = input$logK, H = input$H, fH = input$fH),
      n_injections = input$n_injections,
      v_total = input$v_total,
      c_m = input$c_m,
      c_s = input$c_s
    )
  },
  label = "ITC模拟",
  category = "simulation")
  
  simulated_data(result$result)
})
```

## 🎯 最佳实践

### ✅ DO（推荐做法）

1. **为重要操作添加监控**
   - 拟合计算
   - 文件读写
   - 复杂模拟
   - 用户交互的关键路径

2. **使用描述性标签**
   ```r
   label = "ITC数据拟合（1:1模型）"  # 好
   label = "operation1"              # 差
   ```

3. **选择合适的类别**
   - `"fitting"` - 拟合
   - `"simulation"` - 模拟
   - `"data_io"` - 数据读写
   - `"calculation"` - 计算
   - `"optimization"` - 优化

4. **定期查看性能报告**
   ```r
   # 每周运行一次
   generate_perf_report()
   analyze_performance()
   ```

### ❌ DON'T（避免做法）

1. **不要监控微小操作**
   ```r
   # 不好：太小的操作
   perf_monitor({ x <- 1 + 1 }, label = "加法")
   ```

2. **不要在循环中使用**
   ```r
   # 不好：会产生大量记录
   for (i in 1:1000) {
     perf_monitor({ ... }, label = "循环操作")
   }
   ```

3. **不要忘记配置阈值**
   ```r
   # 根据应用场景调整
   set_perf_config(
     slow_threshold = 1.0,      # Shiny 应用：1秒
     # slow_threshold = 10.0,   # 批处理：10秒
   )
   ```

## 🔧 配置建议

### 开发环境

```r
set_perf_config(
  enabled = TRUE,
  log_level = "detailed",      # 详细记录
  slow_threshold = 0.5,        # 0.5秒即提示
  console_output = TRUE,       # 在控制台显示
  auto_log_slow = TRUE
)
```

### 生产环境

```r
set_perf_config(
  enabled = TRUE,
  log_level = "summary",       # 仅记录重要操作
  slow_threshold = 2.0,        # 2秒以上提示
  console_output = FALSE,      # 不在控制台显示
  auto_log_slow = TRUE
)
```

### 临时禁用

```r
# 如果需要临时禁用（如性能测试时）
set_perf_config(enabled = FALSE)

# 测试完成后重新启用
set_perf_config(enabled = TRUE)
```

## 🆘 常见问题

### Q: 性能监控会影响程序速度吗？

A: 影响很小（< 1ms），对于耗时操作（> 100ms）几乎可忽略。

### Q: 如何查看性能日志文件？

A: 日志保存在 `performance.log`：

```bash
# 查看最近 20 条
tail -n 20 performance.log

# 持续监控
tail -f performance.log
```

### Q: 如何清除旧记录？

A: 
```r
# 清除所有记录
clear_perf_records()

# 清除特定类别
clear_perf_records(category = "test")
```

### Q: 如何查看最慢的操作？

A:
```r
# 获取耗时超过1秒的操作
slow_ops <- get_perf_records(min_elapsed = 1.0)

# 查看最近的10个慢操作
recent_slow <- get_perf_records(min_elapsed = 1.0, last_n = 10)
```
