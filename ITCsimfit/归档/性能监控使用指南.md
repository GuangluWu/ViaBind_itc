## 概述

性能监控模块 (`R/performance.R`) 提供了一套完整的性能监控、日志记录和分析工具，帮助识别和优化应用程序中的性能瓶颈。

### 主要特性

- ✅ **自动性能监控**：包装代码块，自动测量执行时间和内存使用
- ✅ **智能日志记录**：根据配置自动记录慢操作和错误
- ✅ **批量监控**：支持多步骤操作的性能监控
- ✅ **性能分析**：生成性能报告和优化建议
- ✅ **函数包装器**：轻松为现有函数添加性能监控
- ✅ **灵活配置**：可自定义阈值、日志级别等

## 快速开始

### 1. 基础使用

```r
# 监控单个操作
result <- perf_monitor({
  # 你的代码
  data <- read.csv("data.csv")
  process_data(data)
}, label = "数据加载和处理", category = "data_io")

# 获取结果和性能信息
processed_data <- result$result
time_taken <- result$elapsed  # 秒
```

```r
# 在 server.R 中
observeEvent(input$run_fitting, {
```

## 核心功能

### 1. `perf_monitor()` - 单个操作监控

最常用的函数，用于监控单个代码块的性能。

```r
perf_monitor(
  expr,              # 要执行的表达式
  label = "操作名称",  # 操作标签
  category = "分类",   # 操作类别（如 "fitting", "simulation"）
  auto_log = TRUE,   # 是否自动记录到日志
  details = list()   # 额外的详细信息
)
```

**示例：监控数据加载**

```r
result <- perf_monitor({
  data <- read.csv("large_file.csv")
  data
}, 
label = "加载大文件", 
category = "data_io",
details = list(file_size = "10MB", rows = 10000))

cat(sprintf("数据加载耗时: %.3f 秒\n", result$elapsed))
```

### 2. `perf_monitor_batch()` - 批量监控

用于监控多个步骤的操作流程。

```r
results <- perf_monitor_batch(
  steps = list(
    list(expr = quote({ step1() }), label = "步骤1"),
    list(expr = quote({ step2() }), label = "步骤2"),
    list(expr = quote({ step3() }), label = "步骤3")
  ),
  overall_label = "完整流程",
  category = "workflow"
)

# 获取各步骤结果
step1_result <- results$results[[1]]
step2_result <- results$results[[2]]

# 查看各步骤耗时
print(results$summary$step_timings)
```

**示例：监控拟合流程**

```r
fitting_results <- perf_monitor_batch(
  steps = list(
    list(
      expr = quote({
        validate_input_data(input_data)
      }), 
      label = "数据验证"
    ),
    list(
      expr = quote({
        DEoptim::DEoptim(objective_fn, lower, upper, control = de_control)
      }), 
      label = "全局优化(DE)"
    ),
    list(
      expr = quote({
        optim(par_init, objective_fn, method = "L-BFGS-B", lower = lower, upper = upper)
      }), 
      label = "局部优化(L-BFGS-B)"
    ),
    list(
      expr = quote({
        calculate_statistics(fitted_params)
      }), 
      label = "统计分析"
    )
  ),
  overall_label = "ITC拟合完整流程",
  category = "fitting"
)

cat(sprintf("总耗时: %.2f 秒\n", fitting_results$summary$total_elapsed))
```

### 3. `with_perf_monitor()` - 函数包装器

为现有函数添加性能监控，无需修改函数内部代码。

```r
# 原始函数
calculate_itc_curve <- function(params, v_inj, c_m, c_s) {
  # ... 计算逻辑 ...
}

# 添加性能监控
calculate_itc_curve_monitored <- with_perf_monitor(
  calculate_itc_curve,
  label = "ITC曲线计算",
  category = "calculation"
)

# 使用包装后的函数（自动监控）
result <- calculate_itc_curve_monitored(params, v_inj, c_m, c_s)
```

### 4. 性能记录查询

```r
# 获取所有记录
all_records <- get_perf_records()

# 筛选特定类别
fitting_records <- get_perf_records(category = "fitting")

  category = "fitting",
```

```r
# 清除所有记录
clear_perf_records()

# 清除特定类别的记录
clear_perf_records(category = "test")
```

## 配置选项

### 查看当前配置

```r
print(PERF_CONFIG)
```

### 修改配置

```r
set_perf_config(
  enabled = TRUE,              # 是否启用性能监控
  log_level = "detailed",      # 日志级别: "none", "summary", "detailed", "all"
  slow_threshold = 1.0,        # 慢操作阈值（秒）
  warning_threshold = 5.0,     # 警告阈值（秒）
  auto_log_slow = TRUE,        # 自动记录慢操作
  console_output = FALSE,      # 是否在控制台打印
  max_records = 1000           # 最大记录数
)
```

### 配置说明

#### 日志级别 (`log_level`)

- **`"none"`**: 不记录任何日志
- **`"summary"`**: 仅记录重要操作（fitting, optimization, simulation）
- **`"detailed"`**: 记录所有监控的操作（推荐）
- **`"all"`**: 记录所有信息，包括中间步骤

#### 阈值设置

- **`slow_threshold`**: 慢操作阈值，超过此时间会标记为 `SLOW`
- **`warning_threshold`**: 警告阈值，超过此时间会触发 warning

**示例：开发环境配置**

```r
# 开发环境：详细记录，较低阈值
set_perf_config(
  enabled = TRUE,
  log_level = "all",
  slow_threshold = 0.5,      # 0.5秒即视为慢
  console_output = TRUE,     # 在控制台显示
  auto_log_slow = TRUE
)
```

**示例：生产环境配置**

```r
# 生产环境：仅记录重要操作和慢操作
set_perf_config(
  enabled = TRUE,
  log_level = "summary",
  slow_threshold = 2.0,      # 2秒以上才视为慢
  console_output = FALSE,    # 不在控制台显示
  auto_log_slow = TRUE
)
```

## 实际应用示例

### 示例 1: 在 `perform_fitting_sync` 中应用

**改进前：**

```r
perform_fitting_sync <- function(data, params, ...) {
  # 拟合逻辑
  result <- DEoptim(...)
  return(result)
}
```

**改进后：**

```r
perform_fitting_sync <- function(data, params, ...) {
  result <- perf_monitor({
    # 步骤1：数据验证
    validated_data <- validate_input_data(data)
    
    # 步骤2：全局优化
    de_result <- DEoptim(objective_fn, lower, upper, control = de_control)
    
    # 步骤3：局部精化
    optim_result <- optim(de_result$optim$bestmem, objective_fn, 
                          method = "L-BFGS-B", lower = lower, upper = upper)
    
    # 返回结果
    list(
      params = optim_result$par,
      value = optim_result$value,
      convergence = optim_result$convergence
    )
  }, 
  label = "ITC数据拟合",
  category = "fitting",
  details = list(
    data_points = nrow(data),
    n_params = length(params)
  ))
  
  # 如果拟合很慢，发出警告
  if (result$elapsed > 10.0) {
    log_warning(sprintf("拟合耗时较长: %.2f 秒", result$elapsed))
  }
  
  return(result$result)
}
```

### 示例 4: 优化算法性能监控

```r
# 在优化函数中详细监控每个阶段
perform_optimization <- function(objective_fn, lower, upper, method = "hybrid") {
  overall_result <- perf_monitor_batch(
    steps = list(
      # 步骤 1: 全局搜索（差分进化）
      list(
        expr = quote({
          perf_monitor({
            de_control <- DEoptim.control(
              itermax = 100,
              NP = max(10 * length(lower), 50)
            )
            DEoptim(objective_fn, lower, upper, control = de_control)
          }, 
          label = "DE优化-迭代",
          category = "optimization",
          auto_log = TRUE)$result
        }),
        label = "全局搜索(DE)"
      ),
      
      # 步骤 2: 局部精化（L-BFGS-B）
      list(
        expr = quote({
          perf_monitor({
            par_init <- de_result$optim$bestmem
            optim(par_init, objective_fn, method = "L-BFGS-B",
                 lower = lower, upper = upper,
                 control = list(maxit = 500))
          },
          label = "L-BFGS-B优化",
          category = "optimization")$result
        }),
        label = "局部精化(L-BFGS-B)"
      ),
      
      # 步骤 3: 计算不确定性
      list(
        expr = quote({
          perf_monitor({
            calculate_parameter_uncertainty(optim_result$par, objective_fn)
          },
          label = "不确定性估计",
          category = "optimization")$result
        }),
        label = "误差分析"
      )
    ),
    overall_label = "混合优化算法",
    category = "fitting"
  )
  
  return(list(
    result = overall_result$results[[2]],  # 返回 L-BFGS-B 结果
    timings = overall_result$summary$step_timings,
    total_time = overall_result$summary$total_elapsed
  ))
}
```

## 最佳实践

### 1. 何时使用性能监控

**✅ 应该使用的场景：**

- 耗时较长的操作（如拟合、优化、大数据处理）
- 用户交互的关键路径（按钮点击后的操作）
- 文件 I/O 操作
- 复杂的计算或模拟
- 需要优化的代码段

**❌ 不需要使用的场景：**

- 极简单的操作（如变量赋值、简单计算）
- 频繁执行的微小操作（会产生大量记录）
- 调试代码（使用 `debug_print` 更合适）

### 2. 选择合适的类别

建议使用以下标准类别：

- `"fitting"` - 数据拟合
- `"optimization"` - 优化算法
- `"simulation"` - 模拟计算
- `"data_io"` - 数据输入输出
- `"calculation"` - 一般计算
- `"visualization"` - 图形生成
- `"validation"` - 数据验证
- `"workflow"` - 完整流程

### 3. 标签命名规范

- 使用描述性的标签，清楚说明操作内容
- 保持标签简洁（10-30 个字符）
- 对于批量操作，使用层次化命名（如 "拟合流程 - 全局优化"）

**良好示例：**

```r
perf_monitor({...}, label = "ITC数据拟合", category = "fitting")
perf_monitor({...}, label = "读取CSV文件", category = "data_io")
perf_monitor({...}, label = "计算残差", category = "calculation")
```

**避免：**

```r
perf_monitor({...}, label = "test", category = "test")          # 太模糊
perf_monitor({...}, label = "operation1", category = "general")  # 无意义
```

### 4. 性能阈值设置

根据应用场景调整阈值：

```r
# 对于交互式应用（如 Shiny）
set_perf_config(
  slow_threshold = 0.5,      # 0.5秒以上即提示
  warning_threshold = 2.0     # 2秒以上警告
)

# 对于批处理应用
set_perf_config(
  slow_threshold = 5.0,       # 5秒以上提示
  warning_threshold = 30.0    # 30秒以上警告
)
```

### 5. 定期清理记录

为防止内存溢出，建议定期清理旧记录：

```r
# 在 Shiny 应用启动时
onSessionEnded(function() {
  # 保存报告

                                            format(Sys.time(), "%Y%m%d_%H%M%S")))

})
```

## 常见问题

### Q1: 性能监控会影响程序性能吗？

A: 性能监控本身有轻微开销（通常 < 1ms），对于耗时操作（> 100ms）影响可忽略不计。如果关注极致性能，可以：

```r
# 生产环境禁用
set_perf_config(enabled = FALSE)

# 或仅监控关键操作
set_perf_config(log_level = "summary")
```

```r
  data.frame(
    timestamp = as.character(r$timestamp),
    label = r$label,
    category = r$category,
    elapsed = r$elapsed,
    memory_used = ifelse(is.na(r$memory_used), 0, r$memory_used)
  )
}))

# 导出为 CSV
```
