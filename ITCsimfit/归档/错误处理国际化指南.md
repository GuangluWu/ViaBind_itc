# 错误处理国际化指南

## 📅 完成日期
2026-01-27

## 📖 概述

本文档说明如何使用新的错误处理国际化功能。该功能将错误消息集成到多语言翻译系统中，提供一致的、本地化的错误提示。

## ✨ 主要特性

1. **错误消息自动翻译**：根据用户语言设置自动显示相应语言的错误消息
2. **标准化错误代码**：使用统一的错误代码（E001-E999）分类错误
3. **友好的错误提示**：提供标题和详细描述两级信息
4. **向后兼容**：仍支持不使用错误代码的传统方式

## 🔧 核心功能

### 1. 错误代码系统

在 `R/constants.R` 中定义了完整的错误代码：

```r
ERROR_CODES <- list(
  # 数据错误 (E001-E099)
  DATA_INVALID = "E001",        # 数据无效
  DATA_MISSING = "E002",        # 数据缺失
  DATA_FORMAT = "E003",         # 数据格式错误
  
  # 参数错误 (E101-E199)
  PARAM_INVALID = "E101",       # 参数无效
  PARAM_OUT_OF_BOUNDS = "E102", # 参数超出范围
  
  # 计算错误 (E201-E299)
  SOLVER_FAILED = "E201",       # 求解器失败
  OPTIMIZATION_FAILED = "E202", # 优化失败
  CONVERGENCE_FAILED = "E203",  # 收敛失败
  
  # 文件错误 (E301-E399)
  FILE_NOT_FOUND = "E301",      # 文件未找到
  FILE_READ_ERROR = "E302",     # 文件读取错误
  FILE_WRITE_ERROR = "E303",    # 文件写入错误
  
  # 系统错误 (E999)
  UNKNOWN_ERROR = "E999"        # 未知错误
)
```

### 3. 辅助函数

#### `get_error_message(error_code, lang_val)`

获取翻译的错误消息：

```r
# 获取错误消息
msg <- get_error_message("E001", "zh")
# 返回: list(title = "数据无效", desc = "数据验证失败，请检查数据格式和内容")

msg_en <- get_error_message("E001", "en")
# 返回: list(title = "Data Invalid", desc = "Data validation failed. Please check data format and content.")
```

#### `handle_error(e, context, show_to_user, log_to_file, lang_val, error_code)`

增强的错误处理函数，支持错误代码和国际化：

```r
# 使用错误代码
tryCatch({
  # 某些可能失败的操作
  validate_data(data)
}, error = function(e) {
  handle_error(e, 
              context = "数据验证", 
              error_code = ERROR_CODES$DATA_INVALID,
              lang_val = lang())
})
```

## 📝 使用示例

### 示例 1: 基本使用

```r
# 在数据验证中使用
validate_input_data <- function(df, lang_val = "zh") {
  tryCatch({
    # 检查数据框
    if(!is.data.frame(df)) {
      stop("输入不是数据框")
    }
    
    if(nrow(df) == 0) {
      stop("数据框为空")
    }
    
    return(TRUE)
  }, error = function(e) {
    handle_error(e, 
                context = "输入数据验证",
                error_code = ERROR_CODES$DATA_INVALID,
                lang_val = lang_val)
    return(FALSE)
  })
}
```

### 示例 2: 在优化中使用

```r
# 在拟合优化中使用
perform_optimization <- function(params, data, lang_val = "zh") {
  tryCatch({
    # 参数验证
    if(any(is.na(params))) {
      stop("参数包含NA值")
    }
    
    # 执行优化
    result <- optim(params, objective_function, ...)
    
    # 检查收敛
    if(result$convergence != 0) {
      stop(paste("优化未收敛，代码:", result$convergence))
    }
    
    return(result)
  }, error = function(e) {
    # 根据错误类型选择错误代码
    if(grepl("NA值", e$message)) {
      error_code <- ERROR_CODES$PARAM_INVALID
    } else if(grepl("未收敛", e$message)) {
      error_code <- ERROR_CODES$CONVERGENCE_FAILED
    } else {
      error_code <- ERROR_CODES$OPTIMIZATION_FAILED
    }
    
    handle_error(e,
                context = "参数优化",
                error_code = error_code,
                lang_val = lang_val)
    return(NULL)
  })
}
```

### 示例 3: 在文件操作中使用

```r
# 在文件读取中使用
read_data_file <- function(filepath, lang_val = "zh") {
  tryCatch({
    # 检查文件是否存在
    if(!file.exists(filepath)) {
      stop(paste("文件不存在:", filepath))
    }
    
    # 读取文件
    data <- read.csv(filepath)
    
    return(data)
  }, error = function(e) {
    # 根据错误类型选择错误代码
    if(grepl("不存在", e$message)) {
      error_code <- ERROR_CODES$FILE_NOT_FOUND
    } else {
      error_code <- ERROR_CODES$FILE_READ_ERROR
    }
    
    handle_error(e,
                context = "读取数据文件",
                error_code = error_code,
                lang_val = lang_val)
    return(NULL)
  })
}
```

### 示例 4: 使用 safe_execute

```r
# safe_execute 的简化用法
result <- safe_execute({
  # 可能失败的操作
  complex_calculation(params)
}, 
context = "复杂计算",
default = NULL,
lang_val = lang())

# 注意: safe_execute 当前不直接支持 error_code 参数
# 如需使用错误代码，请使用 tryCatch + handle_error
```

## 🎯 最佳实践

### 1. 选择合适的错误代码

根据错误类型选择对应的错误代码：

```r
# 数据相关错误
if(is.null(data)) {
  error_code <- ERROR_CODES$DATA_MISSING
} else if(!valid_format(data)) {
  error_code <- ERROR_CODES$DATA_FORMAT
} else {
  error_code <- ERROR_CODES$DATA_INVALID
}

# 参数相关错误
if(param < min_val || param > max_val) {
  error_code <- ERROR_CODES$PARAM_OUT_OF_BOUNDS
} else {
  error_code <- ERROR_CODES$PARAM_INVALID
}

# 计算相关错误
if(solver_status == "failed") {
  error_code <- ERROR_CODES$SOLVER_FAILED
} else if(!converged) {
  error_code <- ERROR_CODES$CONVERGENCE_FAILED
} else {
  error_code <- ERROR_CODES$OPTIMIZATION_FAILED
}
```

### 2. 提供有用的上下文信息

```r
# 好的实践：提供清晰的上下文
handle_error(e, 
            context = "拟合第3次迭代",
            error_code = ERROR_CODES$OPTIMIZATION_FAILED,
            lang_val = lang())

# 不好的实践：上下文太模糊
handle_error(e, 
            context = "错误",
            error_code = ERROR_CODES$OPTIMIZATION_FAILED,
            lang_val = lang())
```

### 3. 在原始错误消息中包含有用信息

```r
# 好的实践：错误消息包含具体信息
if(param_value < 0) {
  stop(sprintf("参数值 %f 不能为负数", param_value))
}

# 然后在 handle_error 中，这条消息会被附加到翻译的错误描述后面
```

### 4. 向后兼容

如果不确定使用哪个错误代码，可以不提供：

```r
# 向后兼容：不使用错误代码
handle_error(e, 
            context = "某个操作",
            lang_val = lang())
# 仍然可以正常工作，只是不会有标准化的错误标题和描述
```

tryCatch({
