# ITCprocessor Phase 4: 开发注释规范

## 1. 适用范围

适用于 `ITCprocessor` 的新增/改动代码，重点覆盖 Step1 解析、基线、积分与桥接导出边界。
策略：仅对新增/改动文件强制。

## 2. 注释类型与字段

### 2.1 模块头注释（必填）

字段固定：

1. `模块职责`
2. `依赖`
3. `对外接口`
4. `副作用`
5. `变更历史`

### 2.2 关键算法注释（核心算法推荐）

字段固定：

1. `算法目标`
2. `输入约束`
3. `数值稳定策略`
4. `失败回退`
5. `复杂度`

### 2.3 I/O 契约注释（边界函数必填）

字段固定：

1. `输入来源`
2. `字段/类型`
3. `单位`
4. `空值策略`
5. `输出保证`

### 2.4 错误语义注释（宿主入口必填）

字段固定：

1. `错误码/类别`
2. `触发条件`
3. `用户可见性`
4. `日志级别`
5. `恢复动作`

## 3. 最小覆盖要求（Phase 4）

1. `app.R`：`[MODULE_HEADER]` + `[ERROR_SEMANTICS]`
2. `R/data_parser.R`：`[MODULE_HEADER]` + `[IO_CONTRACT]`

## 4. 反例

1. 未写单位与空值策略（尤其是 ITC 原始文件解析）。
2. 启动错误处理没有用户可见性定义。
3. 边界函数只写流程，不写字段约束。
